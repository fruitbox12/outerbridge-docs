# Platform Core Dependencies

This document provides an overview of the core dependencies used in the Weave.Services platform, explaining their purposes and how they contribute to the overall functionality of the platform.

## Dependencies

### Core Dependencies

- **@emotion/react**: Used for writing CSS styles with JavaScript, enabling dynamic styling of components within the Weave.Services platform.
- **@emotion/styled**: A styled-components library built on top of Emotion, allowing for the creation of styled React components for a cohesive UI.
- **@hookform/resolvers**: Provides validation resolvers for `react-hook-form`, used for form validation within the platform.
- **@microlink/react-json-view**: A React component for displaying JSON data in a tree format, useful for debugging and visualizing workflow configurations.
- **@mui/material**: A comprehensive library of UI components based on Material Design principles, used for building a consistent and accessible user interface.
- **@radix-ui/react-accordion**: Used for creating accessible and customizable accordion interfaces, enabling collapsible sections in the UI.
- **@radix-ui/react-avatar**: Displays user avatars, enhancing the user profile and identity features.
- **@radix-ui/react-checkbox**: Provides accessible and customizable checkboxes for form inputs.
- **@radix-ui/react-dialog**: Used for creating accessible and customizable dialog boxes for user interactions and confirmations.
- **@radix-ui/react-dropdown-menu**: Creates accessible and customizable dropdown menus for navigation and options.
- **@radix-ui/react-icons**: A collection of icons used throughout the platform for visual consistency.
- **@radix-ui/react-label**: Provides accessible and customizable labels for form elements, ensuring proper association and accessibility.
- **@radix-ui/react-popover**: Used for creating accessible and customizable popover interfaces, providing additional context or options without navigating away.
- **@radix-ui/react-radio-group**: Creates accessible and customizable radio button groups for form selections.
- **@radix-ui/react-scroll-area**: Provides customizable scroll areas, enhancing the user experience with smooth scrolling.
- **@radix-ui/react-select**: Used for creating accessible and customizable select menus for dropdown selections.
- **@radix-ui/react-separator**: Creates accessible and customizable separators to divide content visually.
- **@radix-ui/react-slot**: Provides accessible and customizable slot interfaces for flexible component composition.
- **@radix-ui/react-switch**: Creates accessible and customizable switches for toggling options.
- **@radix-ui/react-tabs**: Used for creating accessible and customizable tab interfaces for organizing content.
- **@radix-ui/react-toggle**: Provides accessible and customizable toggles for boolean options.
- **@radix-ui/react-tooltip**: Used for creating accessible and customizable tooltips for additional information on hover or focus.
- **@tanstack/react-table**: A headless UI library for building flexible and customizable tables, used for displaying data in the platform.
- **axios**: A promise-based HTTP client for making API requests to backend services and external APIs.
- **bcryptjs**: Used for hashing and comparing passwords, ensuring secure authentication processes.
- **brace**: A fork of Ace editor, used for embedding a code editor within the platform for writing and editing code.
- **class-variance-authority**: A utility for creating variants of CSS classes based on dynamic conditions, enhancing the flexibility of styles.
- **clsx**: A utility for conditionally constructing `className` strings, simplifying dynamic class assignments.
- **date-fns**: A modern JavaScript date utility library for manipulating and formatting dates within workflows and logs.
- **dayjs**: A minimalist JavaScript library for parsing, validating, manipulating, and formatting dates, providing an alternative to date-fns.
- **formik**: A library for building and managing forms in React, used for handling user input and form validation.
- **immer**: A library for working with immutable state, simplifying state management in complex applications.
- **jotai**: A primitive and flexible state management library for React, used for managing state within the platform.
- **jotai-immer**: An extension of Jotai for working with Immer, enabling immutable state management.
- **lucide-react**: A collection of simple and customizable SVG icons for React, used throughout the platform for consistent iconography.
- **mongoose**: An Object Data Modeling (ODM) library for MongoDB, used for interacting with the database in backend services.
- **next**: A React framework for building server-side rendering and static web applications, forming the core of the platform's frontend.
- **next-auth**: A library for implementing authentication in Next.js applications, managing user sessions and security.
- **next-themes**: A library for adding theme support to Next.js applications, allowing users to switch between light and dark modes.
- **react**: A JavaScript library for building user interfaces, the foundation of the platform's frontend.
- **react-ace**: A React component for embedding the Ace code editor, used for writing and editing code within the platform.
- **react-day-picker**: A flexible date picker component for React, used for selecting dates within forms and workflows.
- **react-dom**: The entry point to the DOM for React applications, enabling the rendering of components.
- **react-hook-form**: A library for building and managing forms in React, known for its minimal re-renders and excellent performance.
- **react-hot-toast**: A library for displaying customizable toast notifications, used for providing feedback to users.
- **react-palette**: A library for extracting dominant colors from images, used for theming and visual enhancements.
- **react-toastify**: Another library for displaying toast notifications, providing an alternative to react-hot-toast.
- **reactflow**: A library for building node-based workflows and diagrams, used for visualizing and managing step functions and serverless workflows.
- **sharp**: A high-performance image processing library, used for handling images within the platform.
- **tailwind-merge**: A utility for merging Tailwind CSS classes dynamically, simplifying style management.
- **tailwindcss-animate**: A plugin for adding animations to Tailwind CSS, enhancing the visual appeal of the platform.
- **y-webrtc**: A WebRTC provider for Yjs, enabling real-time collaborative editing and synchronization.
- **yjs**: A CRDT (Conflict-free Replicated Data Type) implementation for building collaborative applications, used for real-time data synchronization.
- **yup**: A schema validation library, used for validating form inputs and ensuring data integrity.
- **zod**: Another schema validation library, providing an alternative to Yup with a focus on TypeScript support.
- **zustand**: A small, fast state management library for React, used for managing global state within the platform.

## DevDependencies

- **@mui/icons-material**: A collection of Material Design icons for use with MUI components.
- **@types/bcryptjs**: TypeScript definitions for bcryptjs.
- **@types/node**: TypeScript definitions for Node.js.
- **@types/react**: TypeScript definitions for React.
- **@types/react-dom**: TypeScript definitions for React DOM.
- **eslint**: A tool for identifying and fixing linting issues in JavaScript code.
- **eslint-config-next**: ESLint configuration for Next.js applications.
- **eslint-config-prettier**: Ensures that Prettier and ESLint play nicely together.
- **eslint-plugin-prettier**: Runs Prettier as an ESLint rule.
- **postcss**: A tool for transforming CSS with JavaScript plugins.
- **prettier**: An opinionated code formatter.
- **react-perfect-scrollbar**: A React wrapper for perfect-scrollbar, used for custom scrollbars.
- **tailwindcss**: A utility-first CSS framework for rapidly building custom designs.
- **typescript**: A typed superset of JavaScript that compiles to plain JavaScript.
